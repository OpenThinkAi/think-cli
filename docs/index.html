// concepts

Engrams become memories.
Memories get recalled.
You control what gets promoted.
Tune curation, customize prompts, or pause logging entirely.

[01]

engrams

Raw work events. Every think sync creates an engram — a short-lived signal waiting to be curated. Stored locally in SQLite. Engrams never leave your machine.

$ think sync "shipped auth refactor"
✓ [engineering] engram saved
[02]

cortex

Your brain's memory folder. Each cortex is a local SQLite database for engrams, plus a folder of JSONL files for the consolidated memories that persist. Point the folder at iCloud, Dropbox, or any sync tool you already use, and your brain's memory follows you across machines.

$ think cortex setup --fs ~/Dropbox/think-cortex
✓ Cortex folder: ~/Dropbox/think-cortex
$ think cortex create personal
✓ Created cortex: personal (local + remote)
[03]

curator

AI-powered evaluation. Reads your engrams, weighs them against what's already known, and promotes what matters to long-term memory. Drops the rest.

$ think curate
Evaluating 12 engrams...
✓ 12 evaluated, 3 promoted, 9 dropped
[04]

episodes

For task-oriented agents (review bots, bug fixers) that work across multiple rounds. Tag engrams with an episode key. When curated, they synthesize into a narrative story — not a log, but a recap of what happened and why it mattered.

$ think sync -e "org/repo#42" "found SQL injection"
$ think curate --episode "org/repo#42"
✓ Episode curated: org/repo#42
[05]

recall & sync

Search across memories and local engrams. Sync pushes memories out to the cortex folder and pulls back what other machines you own have written. All reads hit local SQLite — instant and offline.

$ think recall "auth"
Memories (last 14d):
→ Auth refactor shipped...
$ think cortex sync
✓ Pulled 3, pushed 1

// architecture

Local-first, by design. All reads and writes go to local SQLite. Engrams never leave the machine — only curated memories land in the cortex folder. Point that folder at a sync tool you already use and your brain's memory follows you across machines. No server, no relay; the folder is the propagation layer.

+- your laptop ------------------+       +- cortex folder ----------------+       +- another machine --------------+
|                                |       |                                |       |                                |
| engrams -> curator -> memories |------>| JSONL files                    |<------| recall <-- context             |
|                                |       | (synced via iCloud / Dropbox / |       |                                |
|                                |       |  Syncthing / Drive / …)        |       |                                |
+--------------------------------+       +--------------------------------+       +--------------------------------+

// install

Two minutes from zero to persistent agent memory.

  1. Install & initialize

    $npm install -g open-think
    $think init

    Requires Node 22.5+. think init writes auto-logging instructions into your project's CLAUDE.md — your agents start logging work immediately. The curator and summary features use the Claude Agent SDK and require a Claude subscription.

  2. Point cortex at a folder

    $think cortex setup --fs ~/Dropbox/think-cortex
    $think cortex create personal

    A cortex is your brain's memory folder. Pass --fs a path inside any sync tool you already use (iCloud, Dropbox, Google Drive, Syncthing) and the same memories will land on every machine you own. Skip --fs for offline-only.

    Already on a git-backed setup? think cortex setup <git-remote-url> is still supported and existing setups keep working unchanged — --fs is just the recommended path for new ones.

  3. You're done

    $think sync "shipped the feature"
    $think curate
    $think recall "what did I ship?"

    Agents log work with think sync. Curation promotes the important stuff to long-term memory. Recall pulls it back next session. think cortex sync writes memories out to the folder so your other machines can pick them up.

// all commands

think sync <message>           Log a work event
think sync -e <key> <msg>     Log an episode-tagged event
think log <message>            Log a note (--category, --tags)
think list                     List entries (--week, --since, --category)
think summary                  AI summary (--raw for plain text)
think delete                   Soft-delete entries

think cortex setup [--fs <path> | <repo>]   Configure backend (or no args for offline)
think cortex create <name>     Create a cortex
think cortex list              Show all cortexes (local + remote)
think cortex switch <name>     Set active cortex
think cortex current           Show active cortex
think cortex push              Push local memories to remote
think cortex pull              Pull remote memories to local
think cortex sync              Push + pull
think cortex status            Show sync state

think curate                   Run curation (--dry-run to preview)
think curate --episode <key>   Curate episode into narrative memory
think curate --consolidate     Compress older memories into summary
think monitor                  Show promoted vs dropped engrams
think recall <query>           Search memories + engrams
think memory                   Show memories (--history for timeline)
think pull <cortex>            Read memories from another cortex you have

think curator edit             Edit personal curator guidance
think curator show             Show current guidance
think pause                    Suppress engram creation
think resume                   Re-enable engram creation

think migrate-data             Import git memories into local SQLite
think init                     Set up CLAUDE.md for auto-logging
think export                   Export entries as sync bundle
think import <file>            Import sync bundle
think audit                    Show sync audit log
think config show              Print configuration
think config set <key> <val>   Update a config value
think update                   Update to latest version
